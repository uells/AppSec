## jinja2
### Об `{% extends %}` и `{% block %}`
**Назначение:** реализовать template inheritance (наследование шаблонов), чтобы вынести общий каркас страницы (layout: head, header, footer, подключение статики) в базовый шаблон и переопределять только вариативные части на конкретных страницах.

#### `{% extends "base.html" %}`

* Объявляет текущий шаблон **дочерним** по отношению к указанному базовому (`base.html`).
* При рендеринге итоговая разметка собирается из **родительского** шаблона, а дочерний шаблон **переопределяет блоки**.
* Практическое правило: `extends` должен идти **самым первым** осмысленным оператором в файле (допустимы только пробелы/комментарии до него), иначе наследование может работать некорректно.

#### `{% block name %} ... {% endblock %}`

* Определяет **именованную секцию** (точку расширения) в родительском шаблоне.
* В дочернем шаблоне блок с тем же именем **переопределяет** содержимое родительского блока (override).
* Это контракт между layout и страницами: layout задаёт структуру, страницы поставляют контент.

**Родитель (`base.html`)**

```html
<title>{% block title %}Site{% endblock %}</title>

<main>
  {% block content %}{% endblock %}
</main>
```

**Дочерний (`index.html`)**

```html
{% extends "base.html" %}

{% block title %}Главная{% endblock %}

{% block content %}
  <h1>Привет</h1>
{% endblock %}
```

#### `{{ super() }}`

* Внутри переопределённого блока позволяет **включить содержимое родительского блока** и дополнить его.
* Используется, когда базовый шаблон содержит “дефолт” или общие элементы, а страница должна лишь расширить их.

```html
{% block title %}Главная — {{ super() }}{% endblock %}
```


### Подключение статики
Реализуется с помощью  `url_for()`:

```html
<link rel="stylesheet" href="{{ url_for('static', filename='css/style.css') }}">
```
